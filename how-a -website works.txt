Topic How a website works

THE FOUR LAYERS

FRONTEND
What the user sees and interacts with
Built with HTML for structure CSS for styling and JavaScript for behaviour
It runs in the browser and controls how the website looks and feels

BACKEND
The part users do not see but everything depends on
It handles authentication business logic and data processing
Example:when you log in the backend checks your password finds your account creates a session and sends a response back

DATABASE
This is where all data is stored permanently
The backend reads from and writes to it but the browser never accesses it directly
Examples include PostgreSQL MySQL MongoDB and Firebase

API
API stands for Application Programming Interface. An API acts as a messenger between the frontend and the backend. It allows different parts of a system to communicate with each other.
For example, when a user submits a login form, the frontend sends the login details to an API. The API passes the information to the backend, which processes it and returns a response.

REQUEST RESPONSE FLOW
The request-response flow describes how information moves between the user, frontend, backend, and database

Example: User Login

The user enters an email and password on the login page.
The frontend sends a request to the backend through an API.
The backend receives the request and checks the information.
The backend queries the database to find the user record.
The database returns the requested information.
The backend verifies the password.
The backend sends a response back to the frontend.
The frontend displays the result to the user.

If the login details are correct, the user is logged in and redirected to their dashboard. If the details are incorrect, an error message is displayed.
