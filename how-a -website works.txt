BACKEND MENTORSHIP WEEK 1 NOTES
Topic How a website works

THE FOUR LAYERS

FRONTEND
What the user sees and interacts with
Built with HTML for structure CSS for styling and JavaScript for behaviour
It runs in the browser and controls how the website looks and feels

BACKEND
The part users do not see but everything depends on
It handles authentication business logic and data processing
Example:when you log in the backend checks your password finds your account creates a session and sends a response back

DATABASE
This is where all data is stored permanently
The backend reads from and writes to it but the browser never accesses it directly
Examples include PostgreSQL MySQL MongoDB and Firebase

API APPLICATION PROGRAMMING INTERFACE
The bridge between frontend and backend
It defines how the frontend talks to the backend using endpoints
Common methods include
GET used to read data
POST used to send or create data
PUT used to update data
DELETE used to remove data
Data is usually exchanged in JSON format

REQUEST RESPONSE CYCLE

Browser sends a request to the backend which may interact with the database and then sends a response back to update the page

LOGIN FLOW STEP BY STEP

The user opens the website in a browser and enters the login page.
The browser requests the page from the server and the frontend (login form) is displayed.
The user enters their email and password and clicks the login button.
The frontend sends this information as a request to the backend through an API.
The backend receives the request and processes it. It first checks the data and usually hashes the password for security.
The backend then communicates with the database to find the user’s stored account details.
The database returns the user record if it exists.
The backend compares the stored password with the one provided.
If the details match, the backend creates a session or authentication token.
The backend sends a response back to the frontend confirming login success.
The frontend stores the token and redirects the user to their dashboard.
From this point, every new request from the user includes the token so the system knows they are already logged in.

FLOW:
Browser → Backend API → Database
Database → Backend API → Browser
