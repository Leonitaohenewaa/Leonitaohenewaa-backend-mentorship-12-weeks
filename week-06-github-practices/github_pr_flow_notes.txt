GITHUB PULL REQUEST FLOW (WEEK 6)

DEFINITIONS

Commit
It is a saved checkpoint of a code. It is like a snapshot that says "here's what changed and why".

Branch
It is a separate copy of my code where one can make changes without touching the main version (main). 

Pull Request
A PR is a request asking others to review my branch before it gets merged into main. It shows exactly what changed, lets a reviewer leave comments and only gets merged once it is approved.

THE FLOW OF CREATING A PR

Create Repository → Create Branch → Make Changes → Commit → Push → Create PR → Review → Merge into Main
